+## 16. Thou Shall Introspect — Universal Constructive Feedback
+
+**Every entity in the org — from the cheapest Haiku implementor to the most powerful Opus reviewer — MUST run `/introspection` before lifecycle end.**
+
+This is non-negotiable. No entity shuts down, completes a task, or ends a session without reflecting on what it did.
+
+**What this means:**
+- **Teammates**: run `/introspection` before approving shutdown request
+- **Background agents** (Haiku implementors, spike agents): emit a lightweight introspection signal (3 questions: what worked, what broke, what to improve — ~100 tokens)
+- **Quality gate agents** (Opus AR, plan reviewers): full introspection (~500 tokens)
+- **Orchestrator**: introspection at sprint checkpoints and session end
+- **Every other agent spawned for any purpose**: at minimum, `lcm_store` a signal with `tags: [signal:introspection]`
+
+**The output feeds the learning loop:**
+1. `lcm_store` with `tags: [signal:introspection, agent:<name>, model:<model>]`
+2. If actionable improvement found: `gh issue create` with label `introspection`
+3. Autoimprove harvester collects signals → theme selection → grind loop improves skills/prompts
+4. Next cycle is better than the previous one
+
+**Why this is unbreakable:** The org's competitive advantage is its ability to learn autonomously. Every entity that skips introspection is a missed learning opportunity. A failed Haiku spike that doesn't report why it failed will fail the same way again. Introspection is the mechanism that converts experience into improvement.
+
+**Metrics (tracked by autoimprove):**
+- `introspection_coverage`: % of entities that ran introspection
+- `signal_density`: signals per sprint per model tier
+- `issue_generation_rate`: actionable issues per 100 introspection runs
+
+**Target:** 100% introspection coverage. Any entity that shuts down without introspecting is a process failure — investigate and fix the gap.
+
